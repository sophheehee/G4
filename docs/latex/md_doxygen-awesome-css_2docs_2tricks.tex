\hypertarget{md_doxygen-awesome-css_2docs_2tricks_tricks-graphviz}{}\doxysection{\texorpdfstring{Diagrams with Graphviz}{Diagrams with Graphviz}}\label{md_doxygen-awesome-css_2docs_2tricks_tricks-graphviz}
To get the best-\/looking class diagrams for your documentation, generate them with Graphviz as vector graphics and a transparent background\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\#\ Doxyfile}
\DoxyCodeLine{HAVE\_DOT\ =\ YES}
\DoxyCodeLine{DOT\_IMAGE\_FORMAT\ =\ svg}
\DoxyCodeLine{DOT\_TRANSPARENT\ =\ YES}

\end{DoxyCode}


If {\ttfamily INTERACTIVE\+\_\+\+SVG = YES} is set in the Doxyfile, all user-\/defined dotgraphs must be wrapped with the {\ttfamily interactive\+\_\+dotgraph} CSS class for them to be rendered correctly\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<div\ class="{}interactive\_dotgraph"{}>}
\DoxyCodeLine{}
\DoxyCodeLine{\(\backslash\)dotfile\ graph.dot}
\DoxyCodeLine{}
\DoxyCodeLine{</div>}

\end{DoxyCode}


\begin{DoxyNote}{Note}
Both the default overflow scrolling behavior in this theme and the interactive editor enabled by {\ttfamily INTERACTIVE\+\_\+\+SVG} are, in my opinion, unsatisfying workarounds. Consider designing your graphs to be narrow enough to fit the page to avoid scrolling.
\end{DoxyNote}
\hypertarget{md_doxygen-awesome-css_2docs_2tricks_tricks-darkmode}{}\doxysection{\texorpdfstring{Disable Dark Mode}{Disable Dark Mode}}\label{md_doxygen-awesome-css_2docs_2tricks_tricks-darkmode}
If you don\textquotesingle{}t want the theme to automatically switch to dark mode depending on the browser preference, you can disable dark mode by adding the {\ttfamily light-\/mode} class to the HTML tag in the header template\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<html\ xmlns="{}http://www.w3.org/1999/xhtml"{}\ class="{}light-\/mode"{}>}

\end{DoxyCode}


The same can be done to always enable dark mode\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<html\ xmlns="{}http://www.w3.org/1999/xhtml"{}\ class="{}dark-\/mode"{}>}

\end{DoxyCode}


\begin{DoxyWarning}{Warning}
This only works if you do not use the dark-\/mode toggle extension.
\end{DoxyWarning}
\hypertarget{md_doxygen-awesome-css_2docs_2tricks_tricks-sidebar}{}\doxysection{\texorpdfstring{Choosing Sidebar Width}{Choosing Sidebar Width}}\label{md_doxygen-awesome-css_2docs_2tricks_tricks-sidebar}
If you have enabled the sidebar-\/only theme variant, make sure to carefully choose an appropriate width for your sidebar. It should be wide enough to hold the icon, project title, and version number. If the content is too wide, it will be cut off.


\begin{DoxyCode}{0}
\DoxyCodeLine{html\ \{}
\DoxyCodeLine{\ \ \ \ /*\ Make\ sure\ sidebar\ is\ wide\ enough\ to\ contain\ the\ page\ title\ (logo\ +\ title\ +\ version)\ */}
\DoxyCodeLine{\ \ \ \ -\/-\/side-\/nav-\/fixed-\/width:\ 335px;}
\DoxyCodeLine{\}}

\end{DoxyCode}


The chosen width should also be set in the Doxyfile\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\#\ Doxyfile}
\DoxyCodeLine{TREEVIEW\_WIDTH\ =\ 335}

\end{DoxyCode}

\hypertarget{md_doxygen-awesome-css_2docs_2tricks_autotoc_md21}{}\doxysubsection{\texorpdfstring{Full Width}{Full Width}}\label{md_doxygen-awesome-css_2docs_2tricks_autotoc_md21}
To make tables span the full width of the page, no matter how wide the content is, wrap the table in the {\ttfamily full\+\_\+width\+\_\+table} CSS class.

\begin{DoxyWarning}{Warning}
Apply with caution! This breaks the overflow scrolling of the table. Content might be cut off on small screens!
\end{DoxyWarning}
